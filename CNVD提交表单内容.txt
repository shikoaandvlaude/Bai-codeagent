----------------------------------------------------------------------
【漏洞描述】（建议500字以内）
----------------------------------------------------------------------

2026年3月31日，JeecgBoot 官方通过 issue #9434 修复了 QueryGenerator 类中
的 SQL 注入漏洞，在 installMplus() 和 installAuthMplus() 两个方法中增加了
SqlInjectionUtil.filterContent() 过滤调用。

但同一个类中的第三个方法 installAuthJdbc()（第949-952行）存在完全相同的
代码模式——通过 getSqlRuleValue() 获取规则值后直接拼接到 SQL 字符串中，
却被遗漏，没有加任何过滤。

该方法返回的 SQL 片段最终通过 MyBatis XML 中的 ${permissionSql}（字符串
直替，非预编译参数）拼入最终查询语句，攻击者注入的 SQL 片段会无条件执行。

利用条件：攻击者需具有系统管理员权限，能够配置"数据权限规则"。通过在
数据规则的 ruleValue 字段中写入恶意 SQL（如 UNION SELECT 跨表查询），
即可在任何用户触发相关查询时窃取敏感数据。

----------------------------------------------------------------------
【漏洞成因】
----------------------------------------------------------------------

补丁遗漏。#9434 修复了两个方法，遗漏了第三个同模式方法。

已修补方法（有 filterContent 过滤）:
  - installMplus()        第136-142行
  - installAuthMplus()    第996-1000行

未修补方法（无任何过滤）:
  - installAuthJdbc()     第949-952行

漏洞代码:
  sb.append(sqlAnd + getSqlRuleValue(ruleMap.get(c).getRuleValue()));

注入落地点:
  JeecgDemoMapper.xml 第12行:
  select * from demo where 1=1 ${permissionSql}

该 XML 文件第10行注释原文:
  "根据权限sql查询数据集 20230904 内部写法，调用位置installAuthJdbc，暂不处理sql注入问题"

----------------------------------------------------------------------
【完整调用链】
----------------------------------------------------------------------

1. JeecgDemoServiceImpl.java 第94行:
   String sql = QueryGenerator.installAuthJdbc(JeecgDemo.class);
   return this.baseMapper.queryListWithPermission(page, sql);

2. QueryGenerator.java 第951行:
   sb.append(sqlAnd + getSqlRuleValue(ruleMap.get(c).getRuleValue()));

3. JeecgDemoMapper.xml 第12行:
   select * from demo where 1=1 ${permissionSql}

----------------------------------------------------------------------
【利用方式/PoC】
----------------------------------------------------------------------

前提: 以管理员身份登录系统

步骤:
1. 进入 系统管理 -> 数据权限 -> 数据规则
2. 新增规则，规则类型选 SQL_RULES_COLUMN
3. ruleValue 填入: 1=2 UNION SELECT id, username, id FROM sys_user WHERE 1=1 OR 1=1
4. 保存规则
5. 触发任何使用 installAuthJdbc() 的查询

效果: 原本查询 demo 表的接口，返回了 sys_user 表的用户名等敏感数据。

已通过 H2 数据库模拟环境验证，UNION注入和布尔盲注两种方式均确认有效。
（截图见附件）

----------------------------------------------------------------------
【修复建议】
----------------------------------------------------------------------

QueryGenerator.java 第951行，将:
  sb.append(sqlAnd + getSqlRuleValue(ruleMap.get(c).getRuleValue()));

改为:
  String sqlRule = getSqlRuleValue(ruleMap.get(c).getRuleValue());
  SqlInjectionUtil.filterContent(sqlRule, null);
  sb.append(sqlAnd + sqlRule);

长期建议:
  1. XML 中 ${permissionSql} 改为 MyBatis 动态SQL预编译方式
  2. 审计所有 getSqlRuleValue() 调用点
